# 點選「Class」，再點選「Open New Class」，即可增加一門新課程。
# 點選「Dashboard」，再點選步驟 2 所輸入的課程，即可看到「Schedule」及「My Status」兩個按鈕。
# 點選「Schedule」，再點選「Add Contents」，即可看到 PLWeb 提供的及其他教師分享的教材。
# 點選「TQC<sup>+</sup> Java 6 銜接練習與認證題庫(TQC<sup>+</sup> Java 6)」，即可看到所要「Add」的教材練習。
# 點選欲開放給學生看的教材練習，並將視窗下拉，點選「Add Selected」後，即可設定 Due Date，按「Update」之後，再按「Dashboard」即可看到所加入的教材練習。
# 將這門新加入的課程的課程代號（有10個數字，例如：20xxxxxxxx）告知學生，請學生註冊或加選即可。

===Getting started：===
# 登入帳號後，點選課名，及所要學習的教材練習，即可看到「開始練習」，點選後即可下載編輯器開始練習。如果練習題是「是非」或「選擇」題，那麼不用寫程式，只要按綠色的（Run）按扭，即可開始作答。<br/>如果是程式設計題，則需要先寫程式，再按綠色的（Run）按扭以編譯及執行程式。
# PLWeb 使用學習者執行程式後，所得到的螢幕輸出與標準答案的螢幕輸出（下載的編輯器的左下方的窗格），是否完全相符的方式，來判斷學習者是否做完一個練習。

例如：
編輯器左下方的 Screen Dump 窗格中顯示的是：
  I love Java!
  Java is so good!
而學習者執行程式時的輸出是（love 與 Java 之間多了一個空格）：
  I love  Java!
  Java is so good!
那麼在編輯器右下方的窗格，將會有如下的錯誤信息：
  test (test)
  I love J^
  error: ASCII 74, 32 not match.
  return: -1
  test_error
其中「J^」的意義是：期待比對到一個「J」，但是沒有比對到。而所得到的 ASCII 字元代號是 32。（可以 google ascii 以取得字元的對照）
學習者使用以上的資訊，可以很方便的修改程式，使得輸出與Screen Dump 所顯示的輸出相符。
